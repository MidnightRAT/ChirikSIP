# План: Виправлення GitHub Actions Linux CI

## Проблема

Workflow `build-linux.yml` фейлиться на step "Install build dependencies":

**Причина**: Ubuntu (noble) не має пакета `libpjproject-dev` — PJSIP не входить до Ubuntu repos. Також `qt6-base-dev` та `portaudio19-dev` знаходяться в **universe** repo, який не увімкнений за замовчуванням.

## Що потрібно змінити

### Файл: `.github/workflows/build-linux.yml`

1. **Увімкнути universe repo** перед `apt-get install`:
   ```bash
   sudo add-apt-repository -y universe
   sudo apt-get update
   ```

2. **Замінити `libpjproject-dev`** на збірку PJSIP з source (як у Windows workflow):
   ```bash
   git clone --depth 1 --branch 2.13.1 https://github.com/pjsip/pjproject.git
   cd pjproject && mkdir build && cd build
   cmake .. -DCMAKE_BUILD_TYPE=Release -DCMAKE_INSTALL_PREFIX=/usr
   cmake --build . -j$(nproc)
   sudo cmake --install .
   ```

3. **Інші пакети** (перевірено — є в Ubuntu noble universe):
   - `qt6-base-dev` ✓
   - `portaudio19-dev` ✓
   - `g++` ✓
   - `cmake` ✓
   - `make` ✓
   - `rpmbuild` → пакет `rpm` на Ubuntu

### Порядок дій для build-rpm job:
1. `sudo add-apt-repository -y universe && sudo apt-get update`
2. `sudo apt-get install -y cmake g++ make qt6-base-dev portaudio19-dev rpm`
3. Збірка PJSIP з source (git clone → cmake → build → install)
4. Далі rpmbuild як раніше

### Порядок дій для build-cmake job:
1. Ті самі кроки 1-3
2. Далі cmake build як раніше

## Верифікація

Після пушу змін workflow має пройти зі status `success` на обох jobs:
- `build-rpm` → створить src.rpm та binary.rpm
- `build-cmake` → збере бінарник

Перевірити через: `curl -s "https://api.github.com/repos/MidnightRAT/ChirikSIP/actions/runs?per_page=1"`
